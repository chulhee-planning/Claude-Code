핵심 요구사항

  • 자차 / 용차 / 타사 배차 등 운송 방식별 물류 처리 로직 분기 필요
  • 세금계산서 발행 및 일부 회계 기능 포함 필요 (외부 회계 연계 고려)

유동적인 주문 처리

  • "직선형"이 아닌 "출하전표 중심의 U자형 흐름" 설계
  • 출하 이후 단계에서도 취소 처리 가능 (출하취소, 반품)
  • 주문 취소 시 연계 문서(발주서, 가공요청서, 물동지시 등)에 대한 예외처리 설계

유통 특화 기능

  • 내부물동(미들마일) / 외부물동(라스트마일) 분리 설계
  • 실제 물동(직송, 내부, 용차) + 경유지/업체송장 등 다양한 송장 반영
  • 외주창고 기능
  • 가공/출하 상차지를 기반으로 물동 자동 생성 및 집하

업무 효율성

  • ERP + WMS 통합을 통한 중복입력 제거
  • 출하 가능 상태 아이콘(○, ◐, ●) 및 업무 프로세스 설계
  • ERP 텍스트 항목(운임, 운송조건, 착지 등)을 관리 가능한 데이터로 전환

요구사항 배경 및 문제 정의

기존 ERP 시스템은 10년 이상 윈도우 기반의 패키지 형태로 운영되어 모바일 지원이 불가능하고, 커스터마이징이 제한되어 현업의 다양한 요구사항을 유연하게 반영하지 못했습니다.

또한 웹 기반 WMS를 별도로 구축했으나 ERP와 WMS 간 데이터 중복 입력으로 업무 효율성이 크게 저하되었습니다.

설계 관점의 접근 및 해결 방안

기존 철강 유통업의 업무 프로세스를 전면적으로 재구축하는 것은 현실적으로 어렵다고 판단하여, 현재 업무 프로세스를 기반으로 최적화 방안을 도출하고 개선하는 프로세스 혁신(PI) 작업을 진행하였습니다. 그러나 현업 인터뷰 과정에서 요구사항이 구체적이지 않고 업무 매뉴얼도 없어 지속적으로 요구사항을 파악하고 정리 및 도식화하여 현업과 재확인하는 작업을 반복하였습니다.

마치 '스무고개'와 같은 반복 과정을 거쳐야 했으며, 일정 시점 이후부터는 현업이 미처 이야기하지 않은 업무 케이스를 미리 정리하여 현업의 확인을 받는 방식으로 진행되었습니다. 이 과정에서 현업 담당자로부터 "저보다 업무를 더 잘 아시는 것 같습니다"라는 긍정적인 피드백을 듣기도 하였습니다.

수주 단계에서 최초 발주 및 가공 필요 여부를 입력하여 중복 입력을 방지하고, 키오스크 입력 시 자동으로 물동전표가 생성되는 트리거 기반 자동화 시스템을 설계하여 WMS 및 ERP가 통합된 시스템을 설계하였습니다.

구현 및 개발 세부 사항

  • • 웹 기반 반응형 UI를 설계하여 PC와 모바일 환경 모두 지원 가능하도록 구현하였습니다.
  • • 프로세스 혁신(PI) 결과를 기반으로 ERP와 WMS의 데이터 연동 및 중복 입력 방지 로직을 개발하였습니다.
  • • 업무 상태를 직관적으로 파악할 수 있도록 아이콘(○, ◐, ●) 및 색상 조합을 적용하여 사용자 경험을 향상시켰습니다.
  • • 기존 텍스트 입력 항목을 관리 가능한 데이터베이스 형태로 전환하여 데이터 활용도를 높였습니다.

성과 분석 및 평가

  • • ERP와 WMS의 데이터 중복 입력을 해소하여 업무 처리 시간이 단축되고 데이터 입력 오류가 현저히 감소하였습니다.
  • • 모바일 지원으로 인해 현장 및 외근 근무자의 시스템 접근성이 향상되어 실시간 업무 대응 능력이 강화되었습니다.
  • • 물류 프로세스 자동화로 재고 관리의 정확도와 납기 준수율이 향상되었으며, 고객 만족도 및 내부 업무 효율성 또한 개선되었습니다.
